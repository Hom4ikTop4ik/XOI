# Task D5: Дизайн RESTful веб-сервиса

## Ресурсы географии и справочники

### 1. Получить список всех доступных городов отправления и назначения.
*   **Метод:** `GET`
*   **URL:** `/api/v1/cities`
*   **Описание:** Возвращает уникальный список городов, в которых есть аэропорты.
*   **Формат ответа:**
    ```json
    [
      {"en": "Moscow", "ru": "Москва"},
      {"en": "St. Petersburg", "ru": "Санкт-Петербург"}
    ]
    ```

### 2. Получить список всех доступных аэропортов отправления и назначения.
*   **Метод:** `GET`
*   **URL:** `/api/v1/airports`
*   **Описание:** Возвращает полный список аэропортов.
*   **Формат ответа:**
    ```json
    [
      {
        "airport_code": "SVO",
        "airport_name": {"en": "Sheremetyevo International Airport", "ru": "Шереметьево"},
        "city": {"en": "Moscow", "ru": "Москва"}
      }
    ]
    ```

### 3. Получить список аэропортов в пределах города.
*   **Метод:** `GET`
*   **URL:** `/api/v1/cities/{city_name}/airports`
*   **Описание:** Параметр `{city_name}` может быть на русском или английском.
*   **Пример:** `GET /api/v1/cities/Moscow/airports`

## Расписания

### 5. Расписание входящих рейсов для аэропорта:
> дни недели,
> время прибытия,
> номер рейса,
> пункт отправления.
*   **Метод:** `GET`
*   **URL:** `/api/v1/airports/{code}/schedule/inbound`
*   **Описание:** Показывает регулярный график рейсов, прибывающих в аэропорт.
*   **Поля ответа:** Дни недели, время прибытия (расчетное), номер рейса, пункт отправления.

### 6. Расписание исходящих рейсов для аэропорта:
> дни недели,
> время прибытия,
> номер рейса,
> пункт отправления.
*   **Метод:** `GET`
*   **URL:** `/api/v1/airports/{code}/schedule/outbound`
*   **Описание:** Показывает регулярный график рейсов, вылетающих из аэропорта.
*   **Поля ответа:** Дни недели, время вылета, номер рейса, пункт назначения.

## Поиск маршрутов (Маршрутизация)

### 7. Поиск соединений между двумя точками
*   **Метод:** `POST` (используется POST, так как запрос сложный и содержит фильтры)
*   **URL:** `/api/v1/search`
*   **Тело запроса:**
    ```json
    {
      "from": "SVO",               // Код аэропорта ИЛИ Название города
      "to": "Уфа",                 // Код аэропорта ИЛИ Название города
      "departure_date": "2025-11-20", 
      "booking_class": "Economy",  // 'Economy', 'Comfort', 'Business'
      "max_connections": 1         // 0 (прямой), 1, 2, 3, null (любое)
    }
    ```
*   **Логика поиска:**
    1.  **Разрешение точек:** Если в `from`/`to` указан город, сервис находит все его `airport_code`.
    2.  **Поиск путей:** Рекурсивный поиск (CTE) по таблице `flights` (будущие рейсы).
    3.  **Фильтр даты:** Вылет первого сегмента строго в указанные сутки.
    4.  **Валидация:** Проверка наличия свободных мест на каждом сегменте (используя `class_capacity` и `segments`).
    5.  **Цена:** Подтягивается из `lab_d4_final_price_list`.

## Бронирование и регистрация

### 8. Создание бронирования
*   **Метод:** `POST`
*   **URL:** `/api/v1/bookings`
*   **Описание:** Транзакционное создание брони для одного пассажира на выбранный маршрут.
*   **Тело запроса:**
    ```json
    {
      "passenger_id": "1234 567890",
      "passenger_name": "IVAN IVANOV",
      "booking_class": "Economy",
      "flight_ids": [1052, 3044] // Список ID рейсов (для пересадок)
    }
    ```
*   **Логика:** В рамках одной транзакции проверяется наличие мест, создается запись в `bookings`, затем в `tickets` и в `segments`.

### 6. Онлайн регистрация
*   **Метод:** `POST`
*   **URL:** `/api/v1/check-in`
*   **Тело запроса:**
    ```json
    {
      "ticket_no": "0005432000001",
      "flight_id": 1052
    }
    ```
*   **Логика:** Система находит свободное место (`seat_no`) в самолете для указанного класса и рейса (случайным выбором), создает запись в `boarding_passes` и возвращает назначенный номер места.
